# BarakahPay — Public Roadmap

**Version:** 0.1 (Living document)
**Last updated:** 2026-07-03

This roadmap represents our current intent. Priorities may shift based on user feedback, partner availability, and SCF award status.

---

## Legend

- Done
- In progress
- Planned
- Stretch (subject to capacity)

---

## Phase 0 — Foundation (July 2026)

**Goal:** Establish repo, documentation, community presence.

- GitHub org + repo created
- Business plan + founder execution strategy documented
- README, CONTRIBUTING, SECURITY, CODE_OF_CONDUCT
- Public architecture documentation
- Rust + Soroban CLI dev environment
- Testnet wallet funded
- Discord + Twitter community presence

---

## Phase 1 — Contracts + Landing (Late July 2026)

**Goal:** First working smart contracts on testnet + public landing page.

- Escrow contract v0.1 (Rust, tested, deployed to testnet)
- Institution registry contract v0.1
- Contract deployment scripts
- Landing page live (Next.js, English + Urdu)
- Waitlist form active
- First 100 waitlist signups
- Twitter thread published on Stellar ecosystem
- SCF Discord introduction post

---

## Phase 2 — MVP Frontend (Early August 2026)

**Goal:** Sender + institution portals functional on testnet.

- Sender portal wallet integration
- School selection UI
- Deposit flow (testnet USDC simulation)
- Institution portal with KYB onboarding
- Institution claim submission UI
- On-chain receipt display
- Mobile-responsive PWA

---

## Phase 3 — Traction + Outreach (Mid August 2026)

**Goal:** Real waitlist, real institution LOIs, real SCF community engagement.

- 200+ waitlist signups
- 3-5 signed school LOIs
- Reddit + Facebook diaspora group posts
- SCF Office Hours attended
- 2 warm connections in Stellar ecosystem
- Weekly progress updates on Twitter

---

## Phase 4 — Working Demo + Submission (Late August 2026)

**Goal:** End-to-end testnet demo + SCF Build Award submission.

- Full end-to-end testnet flow demonstrated
- 2-3 minute demo video recorded
- Submission document polished
- Budget breakdown finalized
- SCF interest form submitted
- SCF Discord follow-up post

---

## Phase 5 — Post-Submission Iteration (September 2026)

**Goal:** Iterate on feedback, prepare for evaluation.

- Community feedback incorporation
- Additional institution outreach
- Technical improvements based on early testers
- Panel interview preparation
- Advisor recruitment finalized

---

## Phase 6 — Post-Award Development (Q4 2026 — Assuming Approval)

### Tranche 1 milestones
- Core contracts audited internally
- 15+ schools verified and onboarded
- Sender + institution portals feature-complete on testnet
- Public GitHub documentation site

### Tranche 2 milestones
- MoneyGram Access API integration (sandbox to production)
- 25 schools onboarded
- KYB workflow production-ready
- Multi-language support (English + Urdu)

### Tranche 3 milestones
- Mainnet deployment
- Public SDK for third-party integrators
- Comprehensive monitoring dashboard
- Pakistan corridor soft launch
- First 20+ real transactions
- $2,500+ real volume

---

## Phase 7 — Post-MVP Growth (Q1-Q2 2027)

**Goal:** Scale within Pakistan corridor, prepare corridor expansion.

- 100 schools onboarded
- Recurring payments feature
- First hire: Junior Rust engineer
- Second SCF Build round application (larger scope)
- Bangladesh corridor scoping

---

## Phase 8 — Multi-Corridor Expansion (Q3 2027 onwards)

- Bangladesh corridor launch
- Philippines corridor scoping
- Multi-purpose escrow (medical, utilities, groceries)
- Zakat/charity module
- Series A fundraise preparation

---

## Long-Term Vision (2028+)

- 500+ verified institutions across South Asia
- Multi-corridor operations (Bangladesh, Philippines, Nigeria, Mexico)
- Purpose-of-payment data as first-of-its-kind development data source
- $50M+ annual transaction volume
- Ecosystem of third-party developers building on BarakahPay primitives

---

## How This Roadmap Evolves

This roadmap is public and updated as we learn. Major shifts are documented in `docs/decisions/` as ADRs. Community input is welcome — open a [GitHub Discussion](https://github.com/barakahpay/barakahpay/discussions) to propose changes.
